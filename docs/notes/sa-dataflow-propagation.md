# SA 数据流与故障传播机制

## 1. 背景：矩阵乘法到脉动阵列的映射

以 `Y = X @ W` 为例，其中 `X[M, K]`、`W[K, N]`、`Y[M, N]`。SA 尺寸为 `R × C`（默认 256×256）。

不同数据流将不同维度的元素固定（stationary）在 PE 中，其余维度随数据流流过 PE 阵列。

| 数据流 | 映射规则 | 固定内容 | 流动内容 |
|--------|---------|---------|---------|
| **WS** (Weight Stationary) | `W[k, j] → PE(k%R, j%C)` | 权重 W | 输入 X 横向流动；psum 纵向累积 |
| **OS** (Output Stationary) | `Y[i, j] → PE(i%R, j%C)` | 输出 Y | 输入 X 和权重 W 流过 PE |
| **IS** (Input Stationary) | `X[i, k] → PE(i%R, k%C)` | 输入 X | 权重 W 流过 PE；psum 纵向累积 |

模运算将大矩阵折叠到有限的 SA 尺寸上：如果 `K > R`，多个 k 索引会映射到同一个 PE 行，在不同时间步依次处理。

## 2. WS（Weight Stationary）数据流

### 2.1 正常执行过程

```
时间步 t=0:                             时间步 t=1:
  col0   col1   col2                      col0   col1   col2
row0 W00   W01   W02  ← X[*,0]输入     row0 W00   W01   W02  ← X[*,1]（新输入）
row1 W10   W11   W12                    row1 W10   W11   W12  ← X[*,0]（从row0传来）

  psum: row0计算结果向下传给row1继续累加
```

- 每个周期，输入数据从左向右流动一列
- 每个 PE 执行 `psum += input × weight`（MAC）
- psum 从上向下流动（或保持在本地，取决于具体微架构）

### 2.2 WS + input 故障（以 PE(0,0) 的 bit-flip 为例）

**这是传播最广的模式**。假设 SA=2×2，`X[2,2]`、`W[2,2]`：

```
X = [[1.0, 2.0],      W = [[5.0, 6.0],
     [3.0, 4.0]]           [7.0, 8.0]]

Y_clean = X @ W = [[1×5+2×7, 1×6+2×8],   = [[19, 22],
                   [3×5+4×7, 3×6+4×8]]      [43, 50]]
```

WS 映射：`W[k,j] → PE(k%2, j%2)`，即 `W[0,0]→PE(0,0)`, `W[0,1]→PE(0,1)`, `W[1,0]→PE(1,0)`, `W[1,1]→PE(1,1)`。

现在在 PE(0,0) 的 input register 注入 bit-flip（翻转第 23 位，FP32 符号位 → 值变为 0.5 倍）：

```
受影响：X[:, k] 其中 k % R == 0，即 k=0，X[:, 0] = [1.0, 3.0]
翻转后：X_flipped[:, 0] = [0.5, 6.0]

传播范围：所有 j % C >= 0 的列，即 j=0 和 j=1（全部列）
```

**传播过程：**

```
步骤1：损坏的 X[:,0] 进入 PE(0,0) 和 PE(0,1)（因为 k=0 映射到 row0）
       ──→ 在 PE(0,0)：0.5 × W[0,0] = 0.5 × 5 = 2.5  （原本应为 1×5=5）
       ──→ 在 PE(0,1)：0.5 × W[0,1] = 0.5 × 6 = 3.0  （原本应为 1×6=6）

步骤2：input 数据流向右传播 ──→ PE(0,1) 也收到了损坏的 input
       同时，clean input 继续从 PE(0,0) 向右流到 PE(0,1) 时也用了损坏值

步骤3：psum 纵向累积 ──→ PE(0,0) 的错误 psum(2.5) 向下传给 PE(1,0)
                        PE(0,1) 的错误 psum(3.0) 向下传给 PE(1,1)
                        PE(1,0) 的 clean psum(3×5=15) 加上错误 → 17.5
                        PE(1,1) 的 clean psum(3×6=18) 加上错误 → 21.0
```

**最终效果：**

```
Y_faulty = [[2.5+2×7, 3.0+2×8],   = [[16.5, 19.0],   原本 [[19, 22],
            [0.5×7+15, 0.5×8+18]]     [18.5, 22.0]]         [43, 50]]

ΔY = [[-2.5, -3.0],
      [-24.5, -28.0]]
```

**关键洞察**：PE(0,0) 的一个 input 故障污染了**整个输出矩阵的所有元素**。因为：
- input 数据流向右传播，影响所有后续列（`j_mod >= c_f`）
- psum 向下累积，将错误扩散到所有后续行
- 两部分叠加，导致**全局污染**

**传播规律总结**：
- 越靠左的列（c 越小），影响越多列 → 覆盖面积越大
- 越靠上的行（r 越小），对应越多的 k 索引（模折叠） → 影响的 input 元素越多
- `PE(0,0)` 是 WS+input 模式下影响最大的位置

### 2.3 WS + weight 故障

在 PE(r_f, c_f) 的 weight register 出错：

```
影响范围：仅 W[k, j] 其中 k%R == r_f 且 j%C == c_f
传播：无传播，点故障。一个 PE 损坏一个权重元素。
Y_tilde = X @ W_tilde（标准矩阵乘法，W 中被修改了一个元素）
```

示例：PE(0,0) weight bit-flip → `W[0,0]` 从 5 变为错误值 → 只有 `Y[0,0]` 和 `Y[1,0]` 中涉及 `W[0,0]` 的项受影响（通过 `X[i,0] × W[0,0]`）。

### 2.4 WS + psum 故障

在 PE(r_f, c_f) 的 psum register 出错：

```
影响范围：Y[:, j] 其中 j%C == c_f（单列）
传播：无传播。只影响一个输出列的所有行。
```

由于所有 `j%C == c_f` 的输出列都经过该 PE 列，该列中任何 PE 的 psum 错误会污染该列所有行的输出。

## 3. OS（Output Stationary）数据流

OS 映射：`Y[i,j] → PE(i%R, j%C)`。输出在 PE 中固定累积，input 和 weight 流经阵列。

### 3.1 OS + input 故障

```
影响范围：Y[i, j] 其中 i%R == r_f 且 j%C >= c_f
传播：沿行向右传播（单行内）
```

**例子**：PE(0,0) input bit-flip，flip 整个 X 张量（全部元素），但污染只写入一行输出：

```
in_mask = (i_mod == 0) & (j_mod >= 0)  → Y 的第 0 行所有列
```

完整 X_flipped 参与 `X_flipped @ W` 得到全输出差异 diff，但只有 Y 中 `i%R==0` 的行被污染。在该行内，`j%C >= c_f` 的所有列都被影响。

**与 WS+input 的关键区别**：OS+input 是"一行全污染"，WS+input 是"全局污染"。

### 3.2 OS + weight 故障

```
影响范围：Y[i, j] 其中 i%R >= r_f 且 j%C == c_f
传播：沿列向下传播（单列内）
```

flip 整个 W 张量，但污染只写入 `j%C == c_f` 的列中 `i%R >= r_f` 的行。

### 3.3 OS + psum 故障

```
影响范围：Y[i, j] 其中 i%R == r_f 且 j%C == c_f
传播：无。点故障，只影响映射到该 PE 的单个输出元素。
```

## 4. IS（Input Stationary）数据流

IS 映射：`X[i,k] → PE(i%R, k%C)`。输入在 PE 中固定，权重流过阵列。注意 IS 中 k 维度被映射到 SA 列（不是 N）。

### 4.1 IS + input 故障

```
影响范围：X[i, k] 其中 i%R == r_f 且 k%C == c_f
传播：无。点故障，只影响映射到该 PE 的输入元素。
```

损坏后的 X_tilde 直接参与 `X_tilde @ W`，通过矩阵乘法自然传播到所有输出。

### 4.2 IS + weight 故障

```
影响范围：Y[i, :] 其中 i%R >= r_f（完整行）
传播：沿行向下传播，污染整个输出行
```

flip 整个 W，但 `X` 被 mask 为只有 `i_mod >= r` 且 `k_mod == c` 的非零行参与 `X_masked @ W_diff`。

### 4.3 IS + psum 故障

```
影响范围：Y[i, :] 其中 i%R == r_f（完整行，所有列）
传播：无。污染映射到该 PE 行的所有输出列。
```

K 维度映射到列方向 → psum 沿列方向累积 → 一个 PE 的 psum 错误会污染该行所有输出。

## 5. 总结表

| 数据流 | 故障模式 | 影响范围 | 传播方向 | 传播类型 |
|--------|---------|---------|---------|---------|
| WS | input | Y[全部行, j%C≥c_f] | 右（全列）+ 下（psum） | **全局** |
| WS | weight | W[k%R==r_f, j%C==c_f] | 无 | 点 |
| WS | psum | Y[全部行, j%C==c_f] | 无 | 单列 |
| OS | input | Y[i%R==r_f, j%C≥c_f] | 右（单行内） | 单行 |
| OS | weight | Y[i%R≥r_f, j%C==c_f] | 下（单列内） | 单列 |
| OS | psum | Y[i%R==r_f, j%C==c_f] | 无 | 点 |
| IS | input | X[i%R==r_f, k%C==c_f] | 无 | 点 |
| IS | weight | Y[i%R≥r_f, 全部列] | 下（整行） | 整行 |
| IS | psum | Y[i%R==r_f, 全部列] | 无 | 整行 |

## 7. 故障类型字符串格式

```
{mode}_{op}[_{pos}]
```

- **mode**: `input` / `weight` / `psum` — 哪个寄存器出错
- **op**: `bitflip` / `stuck_0` / `stuck_1` — 错误类型
- **pos**: 位位置（0-31，FP32 解释下的 bit 索引）

示例：`input_bitflip_23` = 翻转 input register 的第 23 位（FP32 符号位），`weight_stuck_1_15` = 将 weight register 第 15 位卡在 1。

对于 BF16 精度，bit 0-15 会自动偏移 +16，因为 BF16 值存储在 FP32 的高 16 位中。
