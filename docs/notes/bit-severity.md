## bit 位代表“影响深度”：错误数值扰动有多大

老师说 bit 位代表影响深度，这个说法是非常合理的。因为 bit flip 首先改变的是一个数值本身，bit 越重要，单次翻转导致的数值偏移越大。这个“深度”不是传播范围，而是**进入计算图之前，错误本身的幅度有多强**。

对于 IEEE 754 FP32，一个数可以写成：
$$
x = (-1)^s \times 2^{e-127} \times (1.m)
$$
其中 sign bit、exponent bits 和 mantissa bits 的影响完全不同。

sign bit 翻转会改变正负号，通常造成接近 $2|x|$ 量级的变化；exponent bit 翻转会改变指数，可能造成数量级变化，通常最危险；mantissa bit 翻转只改变尾数精度，一般影响相对较小，并且越靠高位影响越大。

所以 bit severity 可以定义成：
$$
S_{\text{bit}}(b, x, d) = \frac{|x'_{b,d} - x|}{|x|+\epsilon}
$$
其中 $b$ 是 bit 位置，$d$ 是翻转方向，$x'_{b,d}$ 表示把第 $b$ 位按照方向 $d$ 翻转后的数值。

这里你提到的关键点非常重要：**0→1 和 1→0 的影响确实不同**。例如 exponent bit 从 0 变 1 可能让数值放大很多，而从 1 变 0 可能让数值缩小很多。两者对后续推理的影响未必对称。因此，bit 表最好不要只按 bit index 建一个长度为 32 的表，而应该按：
$$
\text{bit index} \times \text{direction}
$$
建立查表项，也就是最多 64 个条目。

更严谨一点，还可以考虑原始数值分布。因为同一个 bit 翻转对不同 $x$ 的影响也不同。比如 exponent 位翻转在不同指数区间的相对影响不完全一样。因此你可以用校准集中的激活值或权重值统计平均扰动：
$$
S_{\text{bit}}(b,d)
=
\mathbb{E}_{x \sim \mathcal{D}}
\left[
\frac{|x'_{b,d} - x|}{|x|+\epsilon}
\right]
$$
这样就可以提前离线构建一个 bit severity lookup table。这个表只需要计算一次，后面代表性采样时直接查表即可。

这一点可以作为你论文中的一个很强的设计合理性：bit 维度虽然只有 32 位，但它不是随便排序的，而是可以由 IEEE 754 数值语义直接推导和统计得到。
